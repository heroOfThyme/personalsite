# Elementii: From Web Dev to iOS Development - A Case Study

As a web developer primarily working with JavaScript, I wanted to expand my skills into mobile development. Elementii, a periodic table app for iPhone, became my first venture into iOS development using Swift and SwiftUI.

<TechStack primary="Swift, SwiftUI" database="CoreData" architecture="MVVM" deployment="App Store" />

## Project Overview

Elementii serves both educational and reference purposes with three core features:

- An interactive, navigable periodic table
- Detailed information on each element
- Chemistry quizzes to test knowledge

Coming from JavaScript, adapting to Swift's strongly-typed nature required significant mental adjustment. While SwiftUI shares some declarative concepts with React, the underlying paradigms differ considerably.

```swift
// SwiftUI thinking
struct ElementCard: View {
    let element: Element
    let onTap: () -> Void

    var body: some View {
        VStack {
            Text(element.symbol)
                .font(.headline)
            Text(element.name)
                .font(.subheadline)
        }
        .padding()
        .background(Color(element.category.color))
        .cornerRadius(8)
        .onTapGesture {
            onTap()
        }
    }
}
```

## Technical Challenges

### Data Management

The periodic table dataset, while not enormous, contains complex relationships. I chose to:

- Package pre-populated JSON data with the app
- Parse into Swift models at runtime
- Implement a custom caching layer for frequently accessed elements
- Use Core Data for storing user quiz results

### UI Implementation

Creating an interactive periodic table presented unique challenges:

- **Layout precision:** Accurately positioning 118 elements
- **Navigation:** Implementing intuitive zooming and panning
- **Responsiveness:** Ensuring compatibility across different iPhone screens
- **Performance:** Maintaining smooth scrolling with many elements on screen

## Development Timeline

The project took approximately 18 weeks:

1. **Learning Foundation (4 weeks)** - SwiftUI tutorials and small practice apps
2. **Planning and Design (2 weeks)** - Feature definition and UI mockups
3. **Core Development (8 weeks)** - Data layer, UI components, and quiz functionality
4. **Refinement and Testing (4 weeks)** - Performance optimization and user testing

## Key Takeaways

### What Worked Well

- **SwiftUI's Declarative UI** - Accelerated development despite the learning curve
- **MVVM Architecture** - Translated well from web development experience
- **Composable Components** - Breaking UI into reusable pieces improved maintainability

### Challenges Faced

- **Swift Type System** - Adjusting to strict typing initially slowed development
- **iOS Platform Constraints** - Learning platform-specific patterns
- **App Store Process** - Navigating the submission and review process

### Unexpected Benefits

- **Performance Thinking** - iOS development forced me to consider memory and CPU usage
- **UI/UX Considerations** - iOS human interface guidelines provided valuable insights
- **Testing Discipline** - More formal testing requirements improved my overall approach

## Results

Elementii launched successfully with:

- 10,000+ downloads in the first three months
- 4.7/5 average rating
- Featured in "Education" category for two weeks
- Adopted by several high school chemistry departments

## Conclusion
